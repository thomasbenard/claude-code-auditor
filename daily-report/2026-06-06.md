## Articles & Blog Posts

**[Claude Code GitHub Action Flaw Let One Malicious Issue Hijack Repositories](https://thehackernews.com/2026/06/claude-code-github-action-flaw-let-one.html)** by *The Hacker News* — Published June 2 and still heavily circulated. A CVSS 7.8 (High) vulnerability in `checkWritePermissions` unconditionally trusted any actor ending in `[bot]`, enabling attackers to exfiltrate secrets and inject malicious code via prompt injection. Fixed in [`claude-code-action v1.0.94`](https://github.com/anthropics/claude-code-action). Researcher RyotaK (GMO Flatt Security) received a $4,800 bounty.

**[Claude Code's GitHub Actions Vulnerability Lets Attackers Compromise Any Repository](https://cybersecuritynews.com/claude-codes-github-actions-vulnerability/)** by *Cybersecurity News* — Companion write-up with additional technical details on the attack vector: malicious GitHub Apps exploiting the bot-trust bypass to steal OIDC tokens and inject code into downstream repositories.

**[Anthropic Splits Claude Subscriptions: What Changes for Indie Hackers on June 15](https://devtoolpicks.com/blog/anthropic-splits-claude-subscriptions-agent-sdk-credit-june-2026)** by *DevToolPicks* — Clear breakdown of the billing split: Agent SDK calls, `claude -p`, Claude Code GitHub Actions, and third-party agent tools (OpenClaw, Conductor, Zed) move to a dedicated monthly credit pool (Pro: $20, Max 5×: $100, Max 20×: $200). Interactive terminal/IDE sessions are unaffected.

**[Claude Code vs Codex: Which AI Coding Agent Should You Use in 2026?](https://www.firecrawl.dev/blog/claude-code-vs-codex)** by *Firecrawl* — Head-to-head comparison drawing on community data from r/ClaudeCode: Claude Code is faster on simple-to-medium tasks; Codex is slower but more thorough on complex ones.
