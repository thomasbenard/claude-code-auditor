# Claude Code Daily Report — April 1, 2026

> Trending articles, discussions, and resources about Claude Code from the previous day.

## Highlights

March 31 was dominated by a single incident: Anthropic accidentally shipped a JavaScript source map in Claude Code npm package v2.1.88, exposing approximately 512,000 lines of TypeScript source code to the public. The community quickly mirrored the code across GitHub and archiving services, making it effectively permanent despite Anthropic's rollback. Separately, Anthropic acknowledged a rate limit drain bug that has been consuming users' 5-hour session quotas in under 90 minutes since March 23.

## Official Announcements

**[Claude Code v2.1.89 Released](https://github.com/anthropics/claude-code/releases)** — The patch following the leak ships April 1 with hook improvements: a new `defer` permission decision for `PreToolUse` hooks, `CLAUDE_CODE_NO_FLICKER=1` environment variable, named subagents for `@`-mention suggestions, and `MCP_CONNECTION_NONBLOCKING` support.

**[Anthropic Statement on Source Code Exposure](https://www.cnbc.com/2026/03/31/anthropic-leak-claude-code-internal-source.html)** — Anthropic confirmed via CNBC: "No sensitive customer data or credentials were involved or exposed. This was a release packaging issue caused by human error, not a security breach."

**[Anthropic Acknowledges Rate Limit Drain Bug](https://www.theregister.com/2026/03/31/anthropic_claude_code_limits/)** — Anthropic publicly acknowledged that Claude Code has been consuming tokens at abnormally high rates since March 23, with some users exhausting their full 5-hour session limits in under 90 minutes. Anthropic doubled off-peak quotas as a temporary mitigation.
