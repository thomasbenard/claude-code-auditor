# Claude Code Daily Report — March 25, 2026

> Trending articles, discussions, and resources about Claude Code from the previous day.

## Highlights

The biggest story yesterday was Anthropic's launch of **auto mode** for Claude Code — a new permissions mode that uses a classifier to decide which actions are safe to run without user approval, replacing the blunt `--dangerously-skip-permissions` flag. Coverage spread across TechCrunch, 9to5Mac, and Simon Willison's blog, with skeptics questioning whether an AI-based security filter can be truly reliable. Alongside auto mode, Anthropic also rolled out **computer use** for Claude Code on macOS, letting Claude open files, navigate browsers, and run dev tools on your machine remotely.

## Official Announcements

**[Claude Code Gets Auto Mode](https://9to5mac.com/2026/03/24/claude-code-gives-developers-auto-mode-a-safer-alternative-to-skipping-permissions/)** — Anthropic launched auto mode as a research preview for Claude Teams users. Before each tool call, a Claude Sonnet 4.6 classifier reviews the action and blocks potentially destructive operations (force-pushing to git, `curl | bash`, mass cloud deletions) while allowing safe ones (local file edits, read-only API calls, package installs from existing manifests). Enterprise and API access is coming "in the coming days."

**[Claude Can Now Use Your Mac While You're Away](https://www.macrumors.com/2026/03/24/claude-use-mac-remotely-iphone/)** — Anthropic announced computer use for Claude Code and Claude Cowork: when Claude lacks a direct tool for a task, it will point, click, and navigate your screen to complete it — opening files, using the browser, and running dev tools automatically. Available to Claude Pro and Max subscribers as a research preview.
