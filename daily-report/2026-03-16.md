## Official Announcements

**[Claude Code v2.1.76](https://github.com/anthropics/claude-code/releases)** — Shipped March 14 at 01:23 UTC, this release introduces MCP Elicitation: MCP servers can now pause mid-task and request structured input from the user via form fields or browser URLs. Accompanying it are two new hook types (`Elicitation` and `ElicitationResult`) for intercepting and overriding those dialogs. Additional additions: a `-n/--name` CLI flag for naming sessions at startup, a `worktree.sparsePaths` setting for sparse-checkout in large monorepos, a new `/effort` slash command for adjusting model effort level, and a configurable session quality survey (`feedbackSurveyRate`) for enterprise admins. Bug fixes address deferred tools losing input schemas after compaction, plan mode re-approval loops, voice mode keypress handling on Windows, and a circuit breaker for auto-compaction retry loops (capped at 3 attempts).

## Articles & Blog Posts

**[Beyond the Single Prompt: Orchestrating Parallel Context Isolation (PCI) with Claude Code](https://dev.to/rsri/beyond-the-single-prompt-orchestrating-parallel-context-isolation-pci-with-claude-code-f58)** by *Sriramprabhu Rajendran* — Published March 15 on DEV Community. Introduces PCI, a distributed-systems approach to Claude Code where multiple agents work simultaneously on scoped slices of a codebase. Each agent gets a narrow context (e.g., only the DB access layer), reducing hallucinations from "context rot." Coordination happens through a shared `CLAUDE.md` governance file and an `A2A_MESSAGES.log` for inter-agent messaging with file locking. The author reports a multi-day refactoring collapsing to ~4 hours. Worth reading for teams tackling large-scale migrations.
