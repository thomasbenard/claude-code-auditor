# Claude Code Daily Report — June 19, 2026

> Trending articles, discussions, and resources about Claude Code from the previous day.

## Highlights

La journée du 18 juin a été marquée par deux événements majeurs : la sortie de Claude Code v2.1.181 avec des corrections critiques pour la corruption de fichiers sur disques réseau, et le lancement de la mise à jour de juin de Claude Design introduisant l'édition WYSIWYG et l'intégration bidirectionnelle `/design-sync` avec Claude Code. L'Anthropic Blog a également publié un article de référence sur la configuration avancée de Claude Code.

## Official Announcements

**[Claude Code v2.1.181](https://releasebot.io/updates/anthropic/claude-code)** — Version de stabilité majeure publiée le 18 juin. Points clés : nouvelle syntaxe `/config key=value` pour modifier les paramètres directement depuis le prompt (fonctionne en mode interactif, `-p`, et Remote Control) ; correction critique de la corruption de fichiers (zéro-byte / troncature) sur disques réseau et dossiers cloud sync ; correction des crashs au démarrage liés aux configurations corrompues ; amélioration du panel sous-agents (masquage automatique après 30 secondes, limite à 5 lignes avec scroll) ; upgrade du runtime Bun embarqué vers la version 1.4.

**[Claude Design — June Update](https://explainx.ai/blog/claude-design-june-2026-update-design-sync-2026)** — Mise à jour significative du produit Claude Design, annoncée par [@nateparrott](https://x.com/nateparrott/status/2067333017429766481) sur X. La nouveauté phare est l'intégration bidirectionnelle avec Claude Code via la commande `/design-sync` : les équipes peuvent importer leur design system depuis un repo Git ou des fichiers de tokens, et pousser les écrans générés directement dans leur codebase. L'éditeur canvas WYSIWYG permet désormais la manipulation directe des éléments par drag-and-drop. Disponible sur tous les plans payants Claude.
