## Official Announcements

**[Claude Code v2.1.126](https://github.com/anthropics/claude-code/releases)** — Broad release covering smarter model picking, project state management, improved OAuth flows for constrained environments, and Windows/PowerShell stability. Key additions:

- `/model` picker now queries your gateway's `/v1/models` endpoint when `ANTHROPIC_BASE_URL` points to an Anthropic-compatible gateway
- `claude project purge [path]` — deletes all Claude Code state for a project (transcripts, tasks, file history, config); supports `--dry-run`, `-y`, `-i`, `--all`
- `--dangerously-skip-permissions` now bypasses write prompts for `.claude/`, `.git/`, `.vscode/`, and shell config files
- `claude auth login` accepts OAuth codes pasted directly in the terminal — fixes auth in WSL2, SSH, and container environments
- Windows: PowerShell 7 (Microsoft Store, MSI, .NET global tool) now properly detected; treated as primary shell when enabled
- Auto mode: permission stalls now show a red spinner for visibility
- **Bug fixes**: images >2000px now auto-downscale on paste; stream idle timeout errors on Mac sleep resolved; Windows clipboard writes no longer expose content in process args (EDR/SIEM concern); Agent SDK hang on malformed tool names in parallel calls fixed

**[Claude Security Public Beta](https://www.infosecurity-magazine.com/news/anthropic-claude-security-for-ai/)** — Anthropic launched Claude Security (formerly Claude Code Security) in public beta for Enterprise customers. Built on Claude Opus 4.7, it scans codebases for vulnerabilities with scheduled and targeted scans, audit integration, and improved triage tracking.
