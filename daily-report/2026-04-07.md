## Highlights

April 6 was dominated by a critical security patch — Anthropic pushed Claude Code v2.1.90 to fix a command-parser bug that silently bypassed developer-configured deny rules via a 50-subcommand overflow, a flaw traceable directly to the March 31 source code leak. ## Official Announcements

**[Critical Claude Code Bypass Vulnerability Patched](https://letsdatascience.com/news/anthropic-patches-claude-code-bypass-vulnerability-f0cd666c)** — Anthropic released Claude Code v2.1.90 on April 6 to fix a command-parser bug in `bashPermissions.ts` that silently disabled developer-configured `deny` rules whenever a shell command contained more than 50 subcommands. Researchers who reverse-engineered the leaked source code found that the security analysis was capped at 50 entries for performance reasons, letting a malicious 51st subcommand bypass rules entirely — risking SSH key and API token exfiltration in CI environments.
