## Articles & Blog Posts

**[Claude Code is leaking API keys into public package registries](https://bdtechtalks.com/2026/04/27/claude-code-api-token-leak/)** by *BDTechTalks / Lakera* — When developers grant "allow always" permission to terminal commands containing credentials, Claude Code writes those exact command strings (including embedded secrets) to `.claude/settings.local.json`. If that directory isn't `.npmignore`d, credentials ship with the published package. A Lakera scan of 46,500 npm packages found 428 affected packages, with approximately 1-in-13 containing live credentials — and conventional secret-scanning tools miss it because the credentials hide inside AI tool config files rather than source code.

**[Inside Claude Code Architecture: Why Today's Most Advanced AI Coding Agents are Powered by Infrastructure, Not Just Intelligence](https://www.intelligentliving.co/claude-code-architecture-agents/)** — A research team analyzed Claude Code's publicly leaked TypeScript source and found that only ~1.6% of the codebase reflects direct AI decision logic; the remaining ~98.4% is operational infrastructure (tool orchestration, permission systems, session management). A useful framing for understanding why agentic reliability is an engineering problem, not just a model problem.
